Consignes générales

Voici votre examen de C++ ZZ2. Lisez tout le sujet avant de commencer : un certain nombre de tests ont été écrits pour être indépendants ou avec peu d'influence sur les autres.

À la fin de l'examen, vous devez rendre un programme qui compile.

Pour plus de simplicité, vous DEVEZ utiliser trois fichiers en plus de main.cpp : tests.cpp, deviant.hpp et deviant.cpp. Ne changez pas la casse et n'ajoutez pas de fichiers, cela casserait la chaine de compilation et engendrerait une note nulle.

Toutes les déclarations seront dans l'entête, les définitions/implémentations seront dans le cpp.

La ligne de compilation est la suivante :

make

Si besoin, vous pouvez faire un make clean. La première compilation de la bibliothèque Catch est un peu longue.

Ne jamais recommenter un test qui a réussi. Tous les tests décommentés doivent compiler dans le même temps (sauf mention contraire).

Pour garder une trace exécutable de votre programme sans stress supplémentaire :

  • Vous pouvez utiliser GIT. Un rappel des commandes GIT est fait à la fin de l'énoncé. La correction s'interessera uniquement à la version disponible.
  • Si vous préférez faire des copies de répertoires, tous les sous-répertoires de votre $HOME sont analysés et seul le code menant à une meilleure note que les autres est gardé.

DOCUMENTATION CPP

Sujet

Le monde vit des temps difficiles. La population s'est réfugiée dans des cités pour se protéger des déviants, qui, quand ils sont attrapés sont placés dans des centres d'observation en attendant qu'ils soient soignés.

  • Créer une classe Personne qui propose un attribut nom [Personne1, Personne2] et un constructeur par défaut [Personne3]
  • Créer une classe Individu. Les instances bénéficient d'un attribut id incrémenté automatiquement à chaque construction. Une méthode de classe est définie pour aider à compter :-) [Individu1]
  • On dote la classe Individu d'une méthode pour afficher un objet dans un flux [Individu2]
  • Une Personne est en fait un Individu [Personne4] avec la bonne méthode afficher() [Personne5]
  • Les instances d'Individu et de Personne disposent de l'opérateur << [Individu3, Personne6]
  • Les classes Individu et Personne proposent une méthode getType à valeur dans l'enum class définie dans le fichier d'entête donné [Type1, Type2]
  • En fait, si la méthode est appelée sur une instance d'Individu, cela génère une exception - personnalisée ou pas [Exception, ExceptionType]
  • Les personnes habitent des cités de classe Cite. Pour cet exercice, il n'y a qu'une seule cité que vous pouvez définir comme une variable globale ou comme une instance de classe de Cite.
  • Dès qu'une Personne est instanciée, elle est ajoutée à la cité [Ville1, Ville2]. On vous conseille un conteneur de pointeurs :-)
  • Quand on se promène, on peut tomber sur des déviants qui sont en fait des personnes ayant mal "tourné". Les déviants n'appartiennent plus à la cité et on dispose de la date à laquelle ces personnes ont été observées. [Deviant1]
  • Il y une relation d'ordre sur les déviants, elle est fonction de la date d'observation codée comme une chaine de caractères. [Deviant2]
  • Un Deviant est également un Individu avec le bon type [Type3]
  • Les déviants sont placés en observation dans un Centre d'Observations et de Visite des Individus Déviants. Les premières déviants qui peuvent en sortir sont les moins atteints (si on se réfère à la date d'observation). Là encore, un conteneur de pointeurs a été judicieusement choisi. [COVID]

Le conseil de la cité a été durement touché donc il n'est plus en mesure de vous proposer la suite de l'exercice mais sachez qu'un traitement a été trouvé pour soigner les déviants : des biZZous en musique par Porry et Salesses - dans le respect des gestes barrières bien sûr, mais on s'égare.............

Commandes GIT

Préparation du compte (1 fois, au début) :

git config --global user.name "un_nom" git config --global user.email "un_mail"

Préparation du répertoire :

git init

À chaque nouveau fichier:

git add fichier git add *

Sauvegarder / Committer :

git commit -am "commentaire"

git log permet d'avoir la liste des sauvegardes avec un numéro de hachage

Pour revenir en arrière :

git checkout numero_de_hash
OU git revert numero_de_hash
